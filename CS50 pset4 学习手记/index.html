BMP文件

图片是由一个个像素（pixel）排列构成的。对于黑白图像来说，每个像素只需要1bit，就可以创造出一副图像。

对于彩色的图片，每个像素则需要更多的数据来展现不同的颜色。例如，GIF格式支持8位色彩（8-bit color），即每个像素的大小为8bit。BMP,JPEG或者PNG格式则支持24位色彩。而事实上，BMP格式支持1，4，8，16，24，32位色。

本节课程序中首先讨论的是24位色的BMP文件。它的每个像素由R,G,B三种颜色组成，每个元素的取值范围都是[0x00, 0xff]，每种颜色都占8bit。在BMP文件中，除了包含每个像素的色彩信息，在文件的开头也有着一部分元数据，元数据中存储着图片的必要信息。下图所示的就是在BMP format 4.0，Windows 95 上使用的BMP文件的文件头，这也是这节课中我们要处理的信息。

关于文件头的具体含义可以参考BITMAPFILEHEADER和BITMAPINFOHEADER的官方文档。

对于这个定义下的BMP文件，每一行的文件所包含的信息大小，必须是4Byte的整数倍，如果不是4的整数倍，则需要用大小为1Byte的0x00补全，我们称这些1Byte的填充数据为padding。虽然padding不参与图片的构成，但是它会影响图片文件的大小，即biWidth的大小是不包含padding的，但是biSize的大小是要计算padding的。

Resize

在这个练习中，我们要讲图片放大整数倍。最直观的修改方式就是宽和高对应增加n倍，同时复制n次原像素，即可完成放大n倍的任务。但对于整个文件，放大之后会改变的变量有以下5个：

bf.bfSize：图片像素、padding加上文件头所占的大小
bi.biSizeImage：图片像素加padding所占的大小
bi.biWidth：图片宽度
bi.biHeight：图片高度
padding：padding的个数

在确定了放大后会改变的变量之后，我们要考虑的是如何实现复写像素的问题。在横向上复写很简单，使用简单的for循环将写入操作重复操作n便即可。但是对于纵向上的复写就要更复杂一些，这里需要理解指针和文件指针。

指针

指针是C语言中比较难以完全理解但同时功能非常强大的特性之一，因为它可以直接操作内存，完成一些花哨的技巧。

从定义上来说，指针是一个变量，它包含另一个变量的内存中的地址。 我们可以有一个指向任何变量类型的指针。在使用上一般有两个符号，&和*。&variable 是调用某个变量的地址，*variable 则是调用该变量所在地址的值。

假设我们想定义一个交换输入变量的值的函数swap()，定义如下。我们期待得到两个值交换的结果，然而程序实际运行后会发现，两个变量的值没有发生变化。原因是因为，当运行到主函数里面的子函数时，程序会单独开辟出一块新的内存来对这个程序做运算，此时ab的值都是存在新的地址中的。当我们在子函数中得到了结果，回到主函数时，子函数所用的内存块就被抛弃了，这也就导致了主函数的两个变量的值没有交换。

但如果我们使用指针,则可以实现两个变量交换的结果

文件指针

使用指针操作文件，通常有以下的用法

FILE *fopen(const char* filename, const char *mode);

size_t fread(void *ptr, size_t size, size_t nmemb, FILE *stream)；

size_t fwrite(const void * ptr, size_t size, size_t nmemb, FILE * stream);

int fseek(FILE * stream, long offset, int whence);

用法均会在代码中体现，但值得注意的是，当使用指针打开文件时，指针指向的是文件内容的当前位置。
