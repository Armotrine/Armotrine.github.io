Ruby on Rails学习手记（一）

Posted on 2018-06-03 | In Notes

前言：为什么是Ruby on Rails

2017年3月，任天堂的新掌机Nintendo Switch发售。我除了立刻下单买了一台主机之外，还迅速购买了nsprice.com的域名，试图做一个关于switch游戏的价格网站。那时候我在玩PS4，经常逛psnine和metacritic，还妄想着等任天堂出了成就系统之后做一个任版的集讨论、闲聊、成就比较、攻略、询价于一体的综合游戏论坛。摩拳擦掌，我开始在知乎搜索开发网站用什么好，是用世界最好的语言PHP，还是稍微熟悉一些的Flask。

然而，到了2018年4月，不单还没有决定是用python还是PHP，连域名也快要过期了。虽然有一部分原因在于塞尔达和马里奥奥德赛太过精彩，加上实在是不喜欢PHP，但归根到底还是因为自己的懒惰和拖延。于是，就快速选择了Rails这个可以快糙猛地做出原型的框架。就把Python留在科学计算和机器学习的时候再重温和进阶好了。

这个笔记依赖于最新版的https://www.railstutorial.org/，由于也听了一些JHU的Rails课程，所以只记录一些自己觉得重要和容易忘记的要点，以及一些编译中遇到的坑

第一章：从零基础到部署

Cloud9

cloud9是依托于AWS EC2实例的开发环境，致力于在服务器端运行和存储代码，免去开发者重复部署环境和更换变成地点的困扰。

Bundler

Bundler是Rails管理 gems的工具，它通过根目录下的Gemfile文件来生命和控制gems的版本，以保证在运行Rails工程的时候可以找到所有的依赖库，以及确保使用正确版本的gems。

在当与其他人共同开发同一个项目的时候，对方使用bundle install命令后，bundler 会找到Gemfile.lock文件并确保他们的安装环境中的gems和在你的环境中的gems的版本是一样的，这就避免了因版本升级导致的第三方库的依赖问题。

参考资料：https://bundler.io/rationale.html

在编译中学习使用HTTParty的时候，第一次运行Bundler的时候我没有在Gemfile文件中声明所用的gem的名字，而是通过命令行安装，这就导致了在编译Rails工程的时候一直无法找到这个依赖库，快把网上的文档翻烂了才在Bundler的官方文档里查出原因。整个过程大概浪费了1个小时。

第二章：一个初级的玩具级APP

目标

  • 制作一个类似Twitter的微博客，包含用户信息和用户发出的推文

为了达成目标，首先需要来构建所需组件的数据模型，对于用户来说，定义它所需要的信息分别是：用户ID（整数型），用户名称（字符串型），邮件地址（字符串型）。对于用户发出的推文，定义它所需要的信息是推文ID，文本内容和发出这条消息的用户的用户ID。

Model-View-Controller (MVC)

Rails遵循MVC框架，它的设计理念是将数据、前端和后台控制分离，以保证整体应用的稳定运行。

下面这张图就表示了用户在访问host/users时的数据流和信息流去向其中index.html.erb文件为包含ruby代码的前端文件。

  1. The browser issues a request for the /users URL.
  2. Rails routes /users to the index action in the Users controller.
  3. The index action asks the User model to retrieve all users (User.all).
  4. The User model pulls all the users from the database.
  5. The User model returns the list of users to the controller.
  6. The controller captures the users in the @users variable, which is passed to the index view.
  7. The view uses embedded Ruby to render the page as HTML.
  8. The controller passes the HTML back to the browser.

REpresentational State Transfer (REST)

在Rails中，使用RESTful开发代表着大多数组件（例如本章要构建的用户信息和用户的推文）被模块化了，变成了资源（resources），也就意味着他们可以对应到数据库中的CRUD方法和HTTP的请求方法。
